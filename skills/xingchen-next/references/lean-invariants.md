# Lean Invariants

These are the only blocking cross-skill invariants in Lean mode.

## INV-STATE-TRUTH

`project-state.json` outranks every derived artifact, exported pack, review surface, or legacy bundle. Once state exists, no other file becomes parallel canonical truth.

## INV-SOURCE-PACK-TRACE

Every project must register the user's actual input reality in `brief`: `thesis`, `audience`, `goal`, and at least one `brief.sources[]` item. Every factual `brief.claims[]` entry must point through `source_refs` to a proof-eligible source. Lean mode does not use the Extended-only `sources.source_pack` or `sources.asset_manifest` fields.

## INV-PROOF-TRUTH

Every factual claim must trace to a real source. Generated images, generated video, decorative reconstruction, and synthetic UI cannot serve as proof. Proof media must preserve truthful geometry, readable key regions, and its source identity.

## INV-AUDIO-TIMING-TRUTH

Preview and final timing must use real or explicitly accepted audio. Estimated timing is allowed during scripting but must be replaced before `preview-revision` can pass. `script.timeline_revision` identifies the current speech and structural baseline; every scene implementation and preview must match it. A speech, beat, timing, or A-roll structure change makes downstream captions, MG, B-roll, music, and preview evidence stale. Voice, picture, and subtitles must be reviewed together.

## INV-SCENE-COGNITIVE-JOB

Every scene must name one `knowledge_change`, one `dominant_visual`, and one semantic `motion_action`. Motion should reveal, compare, trace, build, transform, hold, or resolve meaning. API level, component count, and continuous movement do not satisfy this rule.

## INV-MOBILE-READABILITY

Essential meaning must survive a phone-sized downsample. Subtitles, thesis labels, important nodes, proof regions, and explicit semantic relationships outrank decorative density. Secondary labels cannot carry the only statement of the argument.

## INV-REAL-PREVIEW-BEFORE-DELIVERY

Entry into final delivery requires two decodable rendered candidates by default for each critical role (`hook`, `hardest-proof`, `payoff`), with a third only for a documented unresolved tradeoff, a selected candidate with a concrete viewing reason, a decodable full preview with accepted audio, a blind full-piece review, a phone-sized readability check, and a real-device playback check. Local transfer is sufficient; platform upload needs explicit authorization. Optional still checkpoints must match the scene type; they cannot substitute for playable candidates. Static boards, code scans, API grep, file extensions, repeated paths, and non-empty fake files are not media evidence.

## INV-FINAL-RENDER-JOB-TRACE

In Lean mode, completing `final-delivery` requires `delivery.status: "delivered"` and a locally decodable video with audio at `delivery.final_path`. Entering final delivery does not require the file yet. Extended projects retain the heavier `render.jobs[]` execution trace.

## INV-REFERENCE-STYLE-CONDITIONAL

A reference video or named visual style is never selected by default. When selected, state must record the source, why it fits the current thesis and audience, the traits to adapt, and an explicit avoid-copying list. Unselected styles add no required fields or lookdev rules.

## INV-SEMANTIC-RELATION-EXPLICIT

Diagram scenes must name the big question, small question, and their relationship directly in state and visible copy. Essential meaning cannot depend on the viewer decoding an abstract loop, arrow, or symbol.

## INV-AGENT-HARNESS-TRACE

From `scene-production` onward, `extensions.agent_harness` must separate runtime policy, workflow skill, bound design-style source, medium strategy, and output grammar. A planned run records successful `route`, `preflight`, and `plan` events. Completing scene production requires the ordered `route -> preflight -> plan -> authorize -> execute -> track -> verify -> commit` trace. Typed generation permissions, job ids, intent fingerprints, server idempotency, reconciliation evidence after ambiguous mutations, resume cursors, charge status, output references, and separate asset-authoring, timeline-placement, and composed-frame receipts must remain inspectable.

## INV-ANTI-PPT-SEMANTIC-MOTION

Every implemented scene must declare a visible before state, causal force, after state, continuity anchor, semantic operators, and boundary evidence. Duration-weighted card topology, repeated-layout, and static-hold ratios must pass the agent-harness gate unless the project explicitly selects an `editorial-card` format with a concrete exception reason. `motion-graphic-only` is valid only when the user or brief selects it; the agent cannot use it as a cheap fallback for a film request. An exception never waives semantic motion or rendered preview evidence.

## INV-SOURCE-LED-FILM

When a source-led or source-bearing mixed-media route is selected, `extensions.source_led_film` must map inspected source events into a contiguous shot plan. Every shot names a viewer question and answer, complexity owner, camera motivation and landing region, sound sync anchor and visible response, continuity anchors, and representation-layer handoff. Source-led work keeps at least half of its duration source-owned; generated connective material cannot silently replace weak or uninspected evidence.

## INV-VOICE-LED-FACELESS

Faceless spoken-knowledge video defaults to `narration-led`: accepted narration is A-roll, not a demand for abundant real footage. `extensions.voice_led_film` must cover every script beat, voice beat, scene, and sound sync point with an explicit visual job. Designed diagrams, motion graphics, typography, B-roll, reusable primitives, and generated metaphor plates are valid picture sources. Real media is required only for factual proof or when the scene genuinely needs a capture; generated material cannot carry proof.
