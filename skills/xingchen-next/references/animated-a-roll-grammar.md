# Animated A-roll Grammar

Use this for faceless narration videos where the designed animation is the primary picture, not decoration over another A-roll.

## Core Idea

The accepted voice owns time. Animated A-roll owns the viewer's eyes. A shot is not a slide; it is a visible state change synchronized to speech.

The default unit is:

`voice beat -> visual state -> semantic motion -> readable hold -> handoff`

## Continuous Objects Beat Slide Changes

Prefer reusing one family of visual objects across a span and changing their state:

- scatter -> group -> resolve;
- question -> branches -> conclusion;
- raw claim -> proof detail -> takeaway;
- before system -> pressure -> changed system;
- hidden cause -> exposed mechanism -> result.

Hard cuts are valid when the idea changes. They are not a substitute for a missing transformation.

## Required State Machine

For animated primary shots, record:

- `opening_state`: what the viewer sees before the beat changes;
- `transform`: the semantic action, such as scatter, group, split, compare, trace, count, reveal, collapse, or resolve;
- `settled_state`: the frame that must read clearly on phone;
- `handoff`: how the outgoing visual prepares the next shot;
- `continuity_anchor`: the object, axis, region, color, or motion direction that survives across the span.

The settled frame is the design authority. Entrance and exit motion serve that frame.

## Sound Lock

Every primary animation needs a sound lock:

- spoken emphasis that triggers a visual change;
- pause that creates a readable hold;
- breath or silence that permits a cut;
- music or accent hit that punctuates the transition.

If the animation could run under any sentence, it is wallpaper, not A-roll.

## Anti-PPT Rule

Reject shots that are only:

- text inside a card;
- unrelated icons appearing one by one;
- a list with generic stagger;
- a static diagram with decorative motion;
- a new full-screen card every few seconds without object continuity.

Allow cards only when the card is the editorial mechanism, such as collecting noisy fragments into one resolved object.

## Reusable Pattern From ChatCut Study

Observed public/account-visible ChatCut MG behavior can be paraphrased as:

`semantic role + full-frame animated object system + staged state changes + readable settled frames + timeline placement`

In the 6-second study clip, one full-frame motion graphic served as the whole A-roll:

1. scattered fragments establish information noise;
2. the same fragments organize around a spine labeled as a process;
3. the fragments collapse away and leave a conclusion frame.

The transferable lesson is not the exact code or style. It is the continuity of objects and the voice-synchronized state machine.
