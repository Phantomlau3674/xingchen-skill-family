# Hybrid Imagegen / Precision Layer Policy

This policy adapts the `steven-superppt-skill` two-layer rule for Xingchen spoken-video production.

## Core Split

Every visual scene has two possible production layers:

1. **Image plate**
   - Use imagegen or sourced imagery for fuzzy, material, emotional, or cinematic things.
   - Examples: atmosphere, background, texture, metaphor object, shock image, hero scene, environment, product-like object, cinematic insert.

2. **Precision layer**
   - Use Remotion, HTML, SVG, Canvas, Three.js, or native code for exact meaning.
   - Examples: text, Chinese subtitles, labels, numbers, charts, tables, arrows, route lines, nodes, axes, UI chrome, evidence highlights, timing, layout grids.

Never burn proof, Chinese text, labels, real UI, charts, or business logic into an AI-generated image. The image may frame the message; code carries the message.

## Spoken Video Translation

For Douyin / voice-led knowledge videos, this becomes:

- **Code-drawn wireframe scenes**: when the visual job is explanation, route logic, data, process, boundary, comparison, node graph, or proof focus. These scenes should be mostly programmatic. The background is solid, gradient, subtle texture, or a very low-contrast plate.
- **Imagegen shock scenes**: when the visual job is emotional hook, metaphor punch, chapter opener, payoff, physical object, environment, or cinematic memory frame. These scenes may be image-heavy, but foreground claims remain code overlays.
- **Hybrid proof scenes**: real screenshot or source pixels first, Remotion focus/scan/callout second, imagegen only as soft supporting plate or surrounding environment.

## Hard Rules

### 1. No Wireframe-On-Busy-Background

Do not place semantic route lines, node graphs, flowcharts, table grids, or dense wireframes on top of a detailed imagegen background.

Allowed only when all are true:

- background has a declared low-contrast safe zone
- scene board names a `precision_layer_safe_area`
- foreground lines pass a contrast check
- proof/subtitles do not overlap the image plate

If the route line or node graph is the explanation, build the whole scene in Remotion/SVG/Canvas instead of treating it as a transparent sticker.

### 2. Imagegen For Plates, Not UI Truth

Use imagegen for:

- background plates
- hero shock images
- chapter plates
- cinematic metaphor scenes
- physical metaphor objects
- material textures
- abstract atmosphere with no readable text

Do not use imagegen for:

- Chinese text
- proof dashboards
- fake screenshots
- real UI evidence
- tables/charts
- labels and callouts
- route lines and node logic
- subtitles

### 3. Code Draws The Argument

If a viewer needs to trust, read, audit, copy, or follow it, draw it with code.

Precision-layer tools:

- Remotion `Sequence`, `spring`, `interpolate`, `Img`, `Video`, `OffthreadVideo`
- SVG paths, masks, arrows, focus frames, scan windows
- Canvas/D3 for data and dense geometry
- Three.js for true 3D camera arguments
- real screenshots/screen recordings for proof, with code overlays

### 4. Imagegen Must Have An Overlay Plan

Every imagegen prompt pack entry must include:

- `visual_job`
- `proof_role`
- `safe_zones`
- `overlay_plan`
- `negative_constraints`
- `acceptance_criteria`
- `expected_output_paths`

Reject an image plate if it competes with the foreground claim, even if it looks beautiful.

## Scene Route Matrix

| Scene job | Default route | Imagegen role | Precision-layer role |
|---|---|---|---|
| hook shock / feed stop | image plate + sparse code overlay | cinematic plate / metaphor / emotional frame | headline, subtitle, small proof handle |
| process explanation | code-native scene | optional soft texture only | routes, nodes, rails, labels, timing |
| proof inspection | real source + code overlay | optional surrounding plate | screenshot, zoom, scan, callout, citation |
| physical metaphor | imagegen object + code motion | object / scene / material | labels, highlights, camera, transitions |
| data explanation | code-native data viz | none or soft plate | chart, axis, numbers, animation |
| payoff / conclusion | image plate or code-native depending on scene | memorable background or object | final claim, stamp, relation lock |

## Asset Kit Implication

The reusable asset library should not be dominated by decorative props.

Prioritize:

- imagegen background plates with real safe zones
- imagegen hero/shock plates for repeatable chapter roles
- imagegen physical metaphor objects only when they carry a recurring concept
- code-native Remotion components for route, node, proof, scan, caption, table, and diagram systems

Deprioritize:

- standalone lamps, cameras, plants, mugs, or other atmosphere-only cutouts
- generic transparent wireframe PNGs pasted over imagegen backgrounds
- fake UI frames generated by image models

## Lookdev Failure Cases

Fail or route back to visual direction if:

- a wireframe overlay is unreadable because the image plate is busy
- generated background has accidental text or fake UI
- semantic lines/nodes are exported as static PNG stickers rather than scene-native Remotion code
- a decorative imagegen prop becomes the hero while the proof is small
- the image plate has no declared safe zone for subtitles and proof
- a generated hero image is pretty but does not serve the scene's visual job
