This intake records the identity of local Stitch family reference archives. The public repository does **not** redistribute those archives because their source package did not establish public redistribution rights. This file is a reference index only. It must not become a template library, default scene bundle, download index, or source-code import path.

Do not copy archive HTML/CSS, remote image URLs, or literal brand copy into Xingchen projects. Use these archives only to name visual families, extract motion/layout questions, and inspire Remotion-native or HyperFrames-native rewrites that still trace to the current director board.

## What Can Be Absorbed

- family names as discovery cues for director-board visual options
- warm creator-avatar design posture recorded in [creator-avatar-family-brand.md](./creator-avatar-family-brand.md)
- motion grammar questions such as flow, comparison, diffusion, attention, manifold, graph, and convergence
- layout questions such as opener, topic chapter, detail proof, data viz, process map, and outro
- component ideas to be rewritten as Remotion-native components or HyperFrames candidates

## What Must Not Be Absorbed

- HTML/CSS source copied into `references/` or project templates
- remote image URLs from the archive
- literal brand copy such as PawScience, Cozy Paws Executive, or course/product claims
- fixed opener, topic, detail, or outro layouts
- mascot placement as a default scene pattern
- proof, UI, chart, subtitle, or claim text generated by the reference component

## Usage Policy

Treat these archives like ReactBits: a reference library, not a renderer family and not a template authority.

Allowed flow:

1. Director board names the scene job, proof boundary, subtitle safe region, and visual gap.
2. Visual compiler may cite a Stitch family name as an inspiration source.
3. Implementation rewrites the idea as a current-state Remotion component, HyperFrames scene, Canvas/SVG motion, or a bounded Wan/AI video plate.
4. Lookdev audits the result against the current director board, not against the archived screenshot.

Any generated or rewritten artifact must still record state traces through `render.scene_motion_specs[]`, `render.hyperframes_candidates[]`, `render.ai_video_candidates[]`, or `render.plugin_adapter_runs[]` as appropriate.
