# Creative Ignition

Run this before proposer. No exceptions.

Purpose:

- force cross-domain thinking before chrome appears
- prevent the default fall into keynote-modern or generic tech aesthetic
- make the content's "way of being seen" explicit before style decisions

## Output Destination

Write the results into `art-direction.md -> ## Ignition`.

## Step 1: Essential Verb

Ask:

- what is this video doing, not merely saying?

Bad:

- `讲 AI 改变创作`

Good:

- `把一个 skill 运行过程当作证据展示给你看`
- `把一组冲突力量正面撞出来`
- `把一份文档里的事实一层层挖出来`

The essential verb should start with an action:

- expose
- compress
- prove
- split
- uncover
- reconstruct
- reveal
- reframe

## Step 2: Candidate Symbolic Worlds

Generate at least 5 candidate worlds.

Hard rules:

- they must cover at least 3 distinct domains
- at least 3 of the 5 may not be software UI worlds
- if all 5 feel like adjacent variants of one interface family, restart

For each world, write:

- `world_name`
- `domain`
- `why_this_content_fits_here`
- `what_the_viewer_thinks_at_second_two`

## Step 3: Counter-Instinct Anchor

Write the easiest cliche version this topic would usually fall into.

Examples:

- AI topic → `saas-keynote`
- data topic → `generic-chart-wall`
- creator topic → `meme-overload`
- insight topic → `kinetic subtitles reading the script`

This counter-instinct anchor must strengthen `anti_reference`, not replace it.

## Step 4: Second-Two Reaction

For each shortlisted world, predict the viewer's second-two reaction.

The reaction must align with the approved `StoryMother` believe target.

Examples:

- terminal world: `诶，这不是 PPT，这是真的在跑东西？`
- archaeology world: `这是在挖什么线索？`
- edit bench world: `这不是讲观点，是把制作过程摊给我看。`

If the reaction does not help the viewer believe the core claim, kill that world before proposer.
